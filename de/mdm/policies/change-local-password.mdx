In diesem Artikel wird erläutert, wie Sie ein lokales Sitzungskennwort über das Primo-Cockpit aktualisieren. Mit dieser Aktion können Administratoren Benutzerkennwörter auf verwalteten Geräten aus der Ferne aktualisieren oder zurücksetzen.

## Voraussetzungen

- Sie sind Administrator Ihres Unternehmensbereichs.
- Das Gerät ist im MDM registriert.
- Das Gerät ist **Online** (eingeschaltet und mit dem Internet verbunden).
- Auf dem Gerät ist eine Administratorsitzung vorhanden.

## Aktualisieren Sie das Passwort von Primo

1. Gehen Sie auf der Gerätedetailseite zur Registerkarte **Benutzer**.
2. Klicken Sie neben der Sitzung, die Sie ändern möchten, auf **Aktion**.
3. Geben Sie das neue temporäre Passwort ein und speichern Sie es.

Sie können den Passwortänderungsstatus auf der Registerkarte **Protokolle** überwachen.

## Verwenden Sie das neue Passwort auf dem Gerät

Sie oder der Benutzer können sich bei der nächsten Sitzungsanmeldung mit den neuen Anmeldeinformationen anmelden.

Das lokale Sitzungskennwort ist jetzt aktualisiert.

## Zurücksetzen der Windows Hello-PIN

Wenn der Windows Hello-PIN-Code nicht funktioniert und der Benutzer sein Passwort nicht verwenden kann, können Sie die PIN zurücksetzen, indem Sie ein PowerShell-Skript ausführen, das den Windows Hello PIN-Ordner (NGC) löscht und den zugehörigen Dienst neu startet:

- Dieses Skript stoppt den NgcCtnrSvc-Dienst.
- Übernimmt den Besitz und legt Berechtigungen für den NGC-Ordner fest.
- Entfernt den Inhalt des NGC-Ordners, um die PIN-Neuerstellung zu erzwingen.
- Startet den NgcCtnrSvc-Dienst neu.

Nach dem Zurücksetzen muss der Benutzer eine neue PIN erstellen.

**Wichtig:**
- Importieren Sie das Skript und führen Sie es mit Fleet aus
- Führen Sie das Skript idealerweise aus, wenn der Benutzer abgemeldet ist.
- Protokolle für diesen Vorgang werden unter `C:\ProgramData\MDM\Logs\Reset-WindowsHelloPIN.log` gespeichert.
