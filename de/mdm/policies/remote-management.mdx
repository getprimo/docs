Geben Sie Ihrem IT-Team mit einem Klick Fernzugriff auf verwaltete Geräte über RustDesk, der automatisch über ein MDM-Profil bereitgestellt wird. Keine manuelle Installation, keine Passwortfreigabe – Sitzungen werden verschlüsselt, im MDM-Audit-Trail protokolliert und sind direkt in der Geräteschublade verfügbar.

## Betriebssystemunterstützung

| macOS | Windows | Linux | iOS / iPadOS | Android |
|-------|---------|-------|--------------|---------|
| ✅ | ✅ | ✅ | | |

## Aktivieren Sie RustDesk in einem MDM-Profil

1. Gehen Sie im Primo-Cockpit zum Abschnitt **Profile**.
2. Wählen Sie ein Profil aus (oder erstellen Sie ein neues).
3. Aktivieren Sie in der Liste der verfügbaren Steuerelemente **RustDesk**.

Der RustDesk-Agent wird dann automatisch auf allen Geräten installiert, die dieses Profil verwenden.

## Starten Sie eine Remote-Verwaltungssitzung

1. Wählen Sie das Gerät aus, mit dem Sie eine Verbindung herstellen möchten, und öffnen Sie dessen Schublade.
2. Scrollen Sie zur Karte „Remotezugriff“ und klicken Sie auf **Sitzung starten**.
   - Sie können die Sitzung in Ihrem Browser oder über die lokale App auf Ihrem Computer ausführen.

Es kann bis zu 30 Sekunden dauern, bis die Sitzung bereit ist. Der Benutzer muss auf seiner Seite nichts tun.

## Auf macOS

<Warning>
  Beim ersten Start der Anwendung wird der Benutzer aufgefordert, **Berechtigungen** wie z. B. die Bildschirmfreigabe zu akzeptieren. Diese Berechtigungen können nicht von MDM bereitgestellt werden und müssen vom Benutzer **genehmigt** werden.
</Warning>

Nachdem der Benutzer die erforderlichen Berechtigungen erteilt hat, starten nachfolgende Sitzungen ohne Benutzerinteraktion.

## Unter Windows

RustDesk wird im Hintergrund installiert und erfordert keine Benutzerinteraktion. Remote-Sitzungen können direkt aus der Geräteschublade gestartet werden, ohne dass eine Einrichtung auf der Geräteseite erforderlich ist.

## Unter Linux

RustDesk wird automatisch auf Linux-Geräten installiert, die im Profil registriert sind. Remote-Sitzungen funktionieren auf die gleiche Weise wie auf anderen Plattformen – starten Sie die Sitzung über die Geräteschublade.

## Sicherheit und Zugang

- **Keine zusätzlichen Passwörter**: Benutzer müssen niemals Codes oder Anmeldedaten senden.
- **Protokolle verfügbar**: Jede Sitzung wird aufgezeichnet, sodass Sie vergangene Aktivitäten überprüfen können.
- **Verschlüsselte Verbindung**: Die Sitzung ist von Anfang bis Ende sicher.
