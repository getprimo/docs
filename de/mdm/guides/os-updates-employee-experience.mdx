## Überwachung des Geräteaktualisierungsstatus

Administratoren können den Betriebssystem-Update-Status für alle verwalteten Geräte direkt im Primo-Cockpit überwachen. Jedes Gerät zeigt einen von drei Update-Status an:

| Status             | Beschreibung                                                                                  |
| ------------------ | -------------------------------------------------------------------------------------------- |
| **Aktuell**     | Auf dem Gerät wird eine Betriebssystemversion ausgeführt, die Ihren konfigurierten Aktualisierungsfristanforderungen entspricht. |
| **Nicht aktuell** | Auf dem Gerät wird eine veraltete Betriebssystemversion ausgeführt und die konfigurierte Frist ist abgelaufen.         |
| **Nicht eingerichtet**      | Für das Gerät wurde keine Betriebssystem-Update-Richtlinie konfiguriert.                                      |

<Note>
  Der Update-Status wird auf der Grundlage Ihrer konfigurierten Fristenrichtlinie für Betriebssystem-Updates berechnet. Geräte mit deaktiviertem MDM zeigen keinen Update-Status an.
</Note>

Sie können die Geräteliste nach Aktualisierungsstatus filtern, um schnell Geräte zu identifizieren, die Aufmerksamkeit erfordern.

## Mitarbeitererfahrung

### macOS 14 und höher

![Screenshot: macOS 14 und höher](/images/notion-import/employee-experience-for-os-updates__img-064__block-0005__1b243d9f.png)


Wenn eine Mindestversion erzwungen wird, sehen die Endbenutzer einmal täglich eine native macOS-Benachrichtigung (DDM). Benutzer können wählen, ob sie die Aktualisierung vor Ablauf der Frist durchführen oder sie für diese Nacht planen möchten. 24 Stunden vor Ablauf der Frist erscheint die Benachrichtigung stündlich und ignoriert „Bitte nicht stören“. Eine Stunde vor Ablauf der Frist erfolgt die Benachrichtigung alle 30 Minuten und anschließend alle 10 Minuten.

Wenn der Host zum Zeitpunkt des Ablaufs der Frist ausgeschaltet war, wird das Update eine Stunde nach dem Einschalten geplant.

Bei macOS-Geräten, die die automatische Geräteregistrierung (Automated Device Enrollment, ADE) verwenden und das Gerät unter der angegebenen Mindestversion ist, muss es während ADE auf die neueste verfügbare Version aktualisiert werden, bevor die Geräteeinrichtung und -registrierung fortgesetzt werden kann.

### macOS 13 und früher

Endbenutzer werden aufgefordert, macOS über Nudge zu aktualisieren.

![Screenshot: Vor macOS 14](/images/notion-import/employee-experience-for-os-updates__img-065__block-0012__1b243d9f.png)


|                             | **> 1 Tag vor dem Stichtag** | **< 1 Tag vor dem Stichtag** | **Frist überschritten** |
| --------------------------- | ---------------------------- | ---------------------------- | -------------------- |
| Fensterfrequenz verschieben      | Einmal täglich um 20:00 Uhr GMT        | Einmal alle 2 Stunden           | Sofort beim Login |
| Der Endbenutzer kann aufschieben          | ✅                            | ✅                            | ❌                    |
| Das Nudge-Fenster kann verworfen werden | ✅                            | ✅                            | ❌                    |

### Windows

Endbenutzern wird empfohlen, Windows über das native Windows-Dialogfeld zu aktualisieren.

|                                      | **Vor Ablauf der Frist** | **Vergangene Frist** |
| ------------------------------------ | ------------------- | ----------------- |
| Der Endbenutzer kann den automatischen Neustart verschieben | ✅                   | ❌                 |

Wenn ein Endbenutzer zum Zeitpunkt des Ablaufs der Frist im Urlaub war, erhält der Endbenutzer eine (konfigurierte) Kulanzfrist, bevor der Host automatisch neu startet.

<Note>
  Für alle oben beschriebenen Szenarien gilt die Frist ab dem „Update-Verfügbarkeitsdatum“ und nicht ab dem Konfigurationsdatum der Richtlinie. Wenn Ihre Mitarbeiter länger als die gewählte Frist eine ältere Version verwenden, werden sie aufgefordert, ihr Betriebssystem sofort zu aktualisieren.
</Note>

## Geringer Speicherplatz

<Warning>
  In allen drei Szenarien muss möglicherweise ein Administrator eingreifen, wenn Aktualisierungen fehlschlagen und der Benutzer keinen Speicherplatz freigeben kann. Keine der Plattformen verwaltet Speicherplatzprobleme alleine, um Updates zu installieren.
</Warning>

### macOS 14 und höher

Der Endbenutzer wird mit einer Systembenachrichtigung dazu gedrängt, das Update zu installieren, und sieht in den Systemeinstellungen, dass nicht genügend Speicherplatz für die Installation des Updates vorhanden ist. Sie müssen selbst Speicherplatz freigeben, um die Aktualisierungsprobleme zu stoppen.

### macOS 13 und früher

Das Nudge-Fenster wird angezeigt und der Benutzer wird zu den Systemeinstellungen weitergeleitet, um das Update herunterzuladen und zu installieren (was fehlschlägt). Das Nudge-Fenster verhält sich wie gewohnt, wenn es nach Ablauf der Frist nicht geschlossen werden kann. Der Endbenutzer muss selbst Speicherplatz freigeben, um das Update zu installieren und das Nudge-Fenster zu schließen.

### Windows

Wenn die Frist eingehalten wird, wird dem Benutzer mitgeteilt, dass er ein Update durchführen muss. Anschließend schlägt das Update fehl und dem Benutzer wird mitgeteilt, dass er Speicherplatz freigeben muss.
