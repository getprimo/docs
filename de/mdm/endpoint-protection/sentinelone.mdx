<Note>
  **Bereitstellungstyp: Automatisch**

  SentinelOne wird **automatisch** über MDM bereitgestellt. Sobald Sie Ihre SentinelOne-Instanz verbinden und ein MDM-Steuerelement erstellen, wird der Agent ohne manuelle Schritte auf alle Zielgeräte übertragen. Es ist keine Benutzerinteraktion erforderlich.
</Note>

In diesem Artikel wird erläutert, wie Sie die SentinelOne-Integration in Primo aktivieren, um den Agent automatisch auf allen oder einem Teil Ihrer Geräteflotte bereitzustellen.

## Voraussetzungen

- Habe Primo MDM
- Sie verfügen über eine SentinelOne-Lizenz (über Primo erworben oder vorhanden)

<Note>
  Benötigen Sie eine SentinelOne-Instanz? Kontaktieren Sie uns für weitere Informationen.
</Note>

## Aktivieren Sie die SentinelOne-Integration

1. Sie können die SentinelOne-Integration über **Einstellungen \> MDM \> Integrationen (alt)** aktivieren.
2. Füllen Sie das Formular mit den folgenden Informationen aus:
   - **SentinelOne-URL**: `https://xxxxxxxxx.sentinelone.net`
   - **Site-ID**:
     - Öffnen Sie in SentinelOne **Richtlinien und Einstellungen\>Sites**.
     - Klicken Sie auf Ihre Site, um die Site-ID anzuzeigen.
   - **Dienstbenutzer-Token**:
     - Öffnen Sie in SentinelOne **Richtlinien und Einstellungen\>Dienstbenutzer**.
     - Erstellen Sie einen neuen Dienstbenutzer (**Neuer Dienstbenutzer**).
     - Wählen Sie Ihre Site aus und weisen Sie die Rolle **Administrator** zu.

## Stellen Sie SentinelOne für Ihre Flotte bereit

Die Bereitstellung von SentinelOne auf den Geräten Ihrer Flotte erfolgt über Ihre MDM-Konfigurationen.

1. Gehen Sie in Primo zu **MDM \> Steuerelemente \> Neues Steuerelement hinzufügen**
2. Wählen Sie die Plattform (verfügbare Plattformen)
3. Aktivieren Sie die Karte **EDR - SentinelOne**
4. Aktualisieren Sie die Ausrichtung, um sie auf die gewünschten Geräte auszurichten
5. Bestätigen

Der SentinelOne-Agent wird automatisch auf allen in der Steuerung enthaltenen Geräten installiert.

## SentinelOne migrieren

Wenn Sie Ihre Endpunkte von einer S1-Konsole auf eine andere migrieren müssen, müssen Sie die folgenden Schritte ausführen:

- Aktualisieren Sie auf Ihrer Fleet-Instanz die Installationsskripts für Sentinel Agent-Pakete mit dem neuen Site-Token.
  - Gehen Sie zu Ihrer Flotteninstanz: https://xxx.mdm.getprimo.com
  - Öffnen Sie die Registerkarte Software
  - Suchen Sie nach „Sentinel“ (SentinelOne, Sentinel Agent usw.)
  - Wählen Sie jedes Paket aus
  - Klicken Sie auf die Schaltfläche Bearbeiten
  - Klicken Sie auf „Erweiterte Optionen“
  - Aktualisieren Sie das Installationsskript
- Wählen Sie auf Ihrer S1-Konsole alle Endpunkte aus, die Sie migrieren möchten, und klicken Sie auf Aktionen \> Endpunktaktionen \> Endpunkte migrieren.
- Geben Sie das neue Site-Token ein und bestätigen Sie es.

## Verfügbare Plattformen

SentinelOne in Primo ist derzeit für die folgenden Plattformen verfügbar:

- **macOS** (arm64) – M1-Chips und mehr
- **Windows** (x64) – Unterstützt Windows Pro
- **Linux**
  - .deb-Pakete (arm64) – Kompatibel mit Ubuntu, Debian und abgeleiteten Distributionen
  - .rpm-Pakete (arm64) – Kompatibel mit RHEL, CentOS, Fedora, SUSE und abgeleiteten Distributionen

Wenn die Plattform oben nicht aufgeführt ist, können Sie SentinelOne dennoch manuell installieren, indem Sie das gewünschte Installationsprogramm von der SentinelOne-Website herunterladen und es manuell mit Fleet installieren.
