---
title: "Erstellen Sie API-Schlüssel für Primo"
description: "Generieren Sie einen Primo API-Schlüssel, um Anforderungen an den Primo REST API zu authentifizieren."
---

Primo API-Schlüssel werden zur Authentifizierung von Anforderungen an den [Primo REST API](https://api.getprimo.com/openapi.json) verwendet. Es gibt zwei Arten von API-Schlüsseln mit jeweils unterschiedlichem Gültigkeitsbereich.

## API-Schlüsseltypen

### Unternehmensschlüssel API

Ein **Firmen-API-Schlüssel** ist auf ein einzelnes Unternehmen beschränkt. Es ermöglicht den Zugriff auf alle Endpunkte, die mit Ressourcen auf Unternehmensebene arbeiten – Geräte, Mitarbeiter, Zubehör, Tickets, SaaS und mehr.

### Arbeitsbereich API-Schlüssel

Ein **Workspace API-Schlüssel** gilt für Ihren gesamten Arbeitsbereich. Es ermöglicht den Zugriff auf Endpunkte auf Arbeitsbereichsebene, z. B. das Auflisten und Anzeigen der Unternehmen in Ihrem Arbeitsbereich.

<Note>
Die meisten API-Endpunkte erfordern einen **Firmen-API-Schlüssel**. Nur die arbeitsbereichsspezifischen Endpunkte (unter `/workspace/`) erfordern einen **Workspace API-Schlüssel**. Überprüfen Sie die API-Referenz auf das Abzeichen an jedem Endpunkt.
</Note>

## Generieren Sie einen Unternehmensschlüssel API

1. Gehen Sie zu **Einstellungen** > **API**.
2. Klicken Sie auf **API-Schlüssel erstellen**.
3. Geben Sie dem Schlüssel einen Namen (z. B. „Meine Integration“).
4. Klicken Sie auf **Erstellen**.
5. Kopieren Sie den Schlüssel sofort – er wird nicht mehr angezeigt.

<Warning>
Bewahren Sie Ihren API-Schlüssel sicher auf. Jeder mit diesem Schlüssel kann über API auf Ihre Primo-Daten zugreifen.
</Warning>

## Verwenden Sie den Schlüssel API

Übergeben Sie den Schlüssel im `Authorization`-Header Ihrer Anfragen:

```bash
curl https://api.getprimo.com/v1/devices \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Einen Schlüssel widerrufen

Um einen Schlüssel zu widerrufen, gehen Sie zu **Einstellungen** > **API**, suchen Sie den Schlüssel und klicken Sie auf **Löschen**.
Sie können auch einen widerrufenen API-Schlüssel löschen.
