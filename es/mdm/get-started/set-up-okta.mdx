Integre Okta como proveedor de autenticación para sus dispositivos administrados utilizando Platform Single Sign-On de Apple. Una vez configurado, los empleados inician sesión en su Mac con sus credenciales de Okta (no se requiere una contraseña local separada) y el SSO se propaga a todas las aplicaciones conectadas a Okta.

## soporte del sistema operativo

| macOS | Windows | Linux | iOS / iPadOS | Android |
|-------|---------|-------|--------------|---------|
| ✅ | | | | |

## En macOS

### Agregue la aplicación Platform Single Sign-On en Okta

1. Vaya a **Consola de administración > Aplicaciones > Aplicaciones > Explorar el catálogo de aplicaciones**
2. Busque **Inicio de sesión único de plataforma para macOS**
3. Haga clic en **Agregar integración**

   > Si aparece el mensaje *Esta función no está habilitada*, comuníquese con su administrador de cuentas Okta para habilitar el acceso al dispositivo Okta (función paga).
4. Abra la aplicación desde su lista de aplicaciones
5. En la pestaña **General**, cambie el nombre de la etiqueta de la aplicación si es necesario
6. En la pestaña **Iniciar sesión**, anote el **ID de cliente** (requerido para la configuración de MDM)
7. En la pestaña **Asignaciones**, asigne la aplicación a los usuarios o grupos relevantes.

### Crea un desafío SCEP en Okta

1. Vaya a **Consola de administración > Seguridad > Integraciones de dispositivos**
2. Asegúrese de que la pestaña **Acceso al dispositivo** esté visible (junto a *Endpoint Management*)

   > Si falta la pestaña, el acceso al dispositivo Okta aún no está habilitado
3. Agregue una **Autoridad de certificación SCEP estática**
4. Tome nota de lo siguiente:
   - **URL del certificado SCEP**
   - **Desafío del certificado SCEP**

### Agregar Okta Verificar a Fleet

1. Desde la Consola de administración Okta, vaya a **Configuración > Descargas**
2. Descargar **Okta Verificar para macOS**
3. Agregue la aplicación a Fleet como software:
   - Simplemente cargue el archivo de la solicitud
   - No habilitar la implementación automática o el autoservicio
   - **Primo ajustará automáticamente la configuración para manejar la implementación**

### Conservar información clave para la configuración de MDM

Asegúrese de guardar los siguientes detalles para los próximos pasos de configuración:

- **Dominio Okta** (por ejemplo, `yourorganization.okta.com`)
- **ID de cliente** (desde la aplicación Platform Single Sign-On)
- **URL del certificado SCEP**
- **Desafío del certificado SCEP**

### Finalizar el despliegue

Para finalizar la implementación de **Okta Platform SSO**, vaya a un perfil: https://app.getprimo.com/profiles

1. Habilite la configuración **SSO de plataforma Okta**
2. Ingrese la información requerida (dominio, ID de cliente, URL de SCEP y desafío)
3. guardar los cambios

Luego, el SSO de la plataforma Okta se activará en todos los dispositivos a los que se dirige el perfil.

<Note>
  La implementación puede tardar varios minutos en completarse, ya que Okta Verify debe estar instalado en todos los dispositivos de destino.
</Note>
