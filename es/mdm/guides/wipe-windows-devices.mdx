El borrado remoto le permite eliminar todos los datos de un dispositivo Windows gestionado a través de Primo. Esto es fundamental cuando un dispositivo se pierde, es robado o se reasigna a otro empleado.

Primo ofrece dos modos de borrado para Windows: **Standard wipe** y **Hard wipe**. Cada uno corresponde a un comando MDM de Microsoft diferente y tiene un comportamiento distinto.

## Compatibilidad del sistema operativo

| macOS | Windows Pro | Windows ARM | Windows Home | Linux | iOS / iPadOS | Android |
|-------|-------------|-------------|--------------|-------|--------------|---------|
| ✅ | ✅ | ✅ | ❌ | ✅ | ✅ | ❌ |

## Standard wipe vs. hard wipe

|  | **Standard wipe** | **Hard wipe** |
|--|-------------------|---------------|
| **Comando Microsoft** | [`doWipe`](https://learn.microsoft.com/en-us/windows/client-management/mdm/remotewipe-csp#dowipe) | [`doWipeProtected`](https://learn.microsoft.com/en-us/windows/client-management/mdm/remotewipe-csp#dowipeprotected) |
| **Qué hace** | Realiza un restablecimiento de fábrica equivalente a **Restablecer este PC** > **Quitar todo** | Limpia completamente el disco interno y sigue reintentando hasta completar el restablecimiento |
| **Puede interrumpirse** | Sí — apagar y encender el dispositivo puede detener el proceso | No — el borrado se reanuda automáticamente después de cualquier interrupción |
| **Si se interrumpe** | El dispositivo intenta volver a su estado anterior al restablecimiento. Si la reversión falla, el dispositivo puede quedar inutilizable | El borrado se reintenta en el siguiente arranque hasta completarse |
| **Riesgo de dispositivo sin arranque** | Bajo — se intenta una reversión en caso de fallo | Mayor — [Microsoft indica](https://learn.microsoft.com/en-us/windows/client-management/mdm/remotewipe-csp#dowipeprotected) que este comando puede dejar el dispositivo sin capacidad de arranque en algunas configuraciones |
| **Ideal para** | Reasignación de dispositivos, renovación rutinaria | Dispositivos perdidos o robados, incidentes de seguridad confirmados |

### Standard wipe (`doWipe`)

El standard wipe envía el comando Microsoft `doWipe`. Es equivalente a ejecutar **Restablecer este PC** > **Quitar todo** desde la configuración de Windows. Todos los datos de usuario, aplicaciones y configuraciones locales se eliminan.

Si el borrado se interrumpe (corte de energía, apagado forzado), Windows intenta volver al estado anterior al restablecimiento. Si la reversión falla, el dispositivo puede quedar inutilizable y será necesario reinstalar Windows.

### Hard wipe (`doWipeProtected`)

El hard wipe envía el comando Microsoft `doWipeProtected`. A diferencia del standard wipe, **sigue reintentando el restablecimiento hasta completarlo** — no puede eludirse apagando y encendiendo el dispositivo.

<Warning>
  Según la [documentación de Microsoft](https://learn.microsoft.com/en-us/windows/client-management/mdm/remotewipe-csp#dowipeprotected), en algunas configuraciones de dispositivos el comando `doWipeProtected` **puede dejar el dispositivo sin capacidad de arranque**. El dispositivo podría requerir una reinstalación manual del sistema operativo usando una unidad USB de recuperación. Use el hard wipe solo cuando no pueda garantizar el acceso físico al dispositivo.
</Warning>

## Cuándo usar cada modo de borrado

| Escenario | Modo recomendado |
|-----------|------------------|
| Dispositivo perdido o robado | **Hard wipe** — asegura que los datos se borren incluso si alguien interrumpe el proceso |
| Offboarding con reasignación del dispositivo | **Standard wipe** — suficiente cuando tiene acceso físico al dispositivo |
| Dispositivo comprometido por malware | **Hard wipe** — garantiza una limpieza completa incluso si el malware interfiere |
| Renovación rutinaria de dispositivos | **Standard wipe** — menor riesgo, recuperación más fácil si algo sale mal |

## Borrar un dispositivo Windows

1. Vaya a **Dispositivos** y localice el dispositivo que desea borrar.
2. Abra la página de detalles del dispositivo.
3. Haga clic en **Acciones** > **Borrar**.
4. Elija **Standard wipe** o **Hard wipe**.
5. Confirme la acción en el cuadro de diálogo de confirmación.

El dispositivo recibe el estado **Borrado pendiente**. Una vez que el dispositivo confirma el comando, el estado se actualiza a **Borrado**.

## Después del borrado

Una vez que un dispositivo ha sido borrado, siga estos pasos antes de volver a inscribirlo:

1. **Reinstalar Windows si es necesario** — un hard wipe limpia completamente el disco, por lo que probablemente necesitará reinstalar el sistema operativo usando una unidad USB de recuperación o arranque por red (PXE). Un standard wipe puede dejar Windows en un estado de restablecimiento utilizable.
2. **Eliminar el host de Fleet** — antes de volver a inscribir el dispositivo, elimine su registro de Fleet. Esto garantiza que las claves de cifrado del disco se depositen correctamente en la nueva inscripción.
3. **Volver a inscribir el dispositivo** — siga su proceso de inscripción estándar para volver a poner el dispositivo bajo gestión.

<Note>
  Si el dispositivo estaba inscrito mediante Windows Autopilot y todavía está registrado en su tenant de Azure/Entra, entrará automáticamente en el flujo de Autopilot en la próxima configuración. Consulte [inscripción zero-touch Windows](/es/mdm/zero-touch/zero-touch-windows) para más detalles.
</Note>

## Limitaciones

- **Windows Home no es compatible** — el borrado remoto requiere Windows Pro, Enterprise o Education. Consulte [gestionar dispositivos Windows Home](/es/mdm/policies/manage-windows-home) para alternativas.
- **El hard wipe puede dejar el dispositivo sin arranque** — en algunas configuraciones de hardware, `doWipeProtected` puede impedir que el dispositivo arranque. Tenga una unidad USB de recuperación preparada.
- **El standard wipe puede fallar en la reversión** — si se interrumpe, el dispositivo puede quedar inutilizable requiriendo una reinstalación manual del sistema operativo.
- **El borrado es irreversible** — una vez iniciado y confirmado por el dispositivo, el borrado no puede cancelarse. Todos los datos se eliminarán permanentemente.
- **Clave de recuperación de BitLocker** — si BitLocker estaba activado, asegúrese de que la clave de recuperación esté respaldada antes del borrado. Después del borrado, la clave anterior ya no es válida.
