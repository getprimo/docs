El borrado remoto le permite eliminar todos los datos de un dispositivo Windows gestionado a través de Primo. Esto es fundamental cuando un dispositivo se pierde, es robado o se reasigna a otro empleado.

## Compatibilidad del sistema operativo

| macOS | Windows Pro | Windows ARM | Windows Home | Linux | iOS / iPadOS | Android |
|-------|-------------|-------------|--------------|-------|--------------|---------|
| ✅ | ✅ | ✅ | ❌ | ✅ | ✅ | ❌ |

## Cómo funciona el borrado en Windows

Primo utiliza las capacidades MDM de Fleet para enviar el comando [`doWipeProtected`](https://learn.microsoft.com/en-us/windows/client-management/mdm/remotewipe-csp#dowipeprotected) de Microsoft al dispositivo.

A diferencia de un borrado estándar (`doWipe`), que puede interrumpirse simplemente apagando y encendiendo el dispositivo, `doWipeProtected` **sigue reintentando el restablecimiento hasta completarlo**. Esto lo convierte en la opción correcta para dispositivos perdidos o robados donde no se puede garantizar el acceso físico.

### Qué hace `doWipeProtected`

1. **Limpia completamente el disco interno** — todos los datos de usuario, aplicaciones y configuraciones locales se eliminan permanentemente.
2. **Persiste ante interrupciones** — si el borrado se interrumpe (corte de energía, apagado forzado), se reanuda automáticamente en el siguiente arranque.
3. **No conserva datos de aprovisionamiento** — el dispositivo vuelve a un estado vacío. Será necesario reinstalar Windows.

<Warning>
  Según la [documentación de Microsoft](https://learn.microsoft.com/en-us/windows/client-management/mdm/remotewipe-csp#dowipeprotected), en algunas configuraciones de dispositivos el comando `doWipeProtected` **puede dejar el dispositivo sin capacidad de arranque**. Esto significa que el dispositivo podría requerir una reinstalación manual del sistema operativo usando una unidad USB de recuperación.
</Warning>

## Cuándo borrar un dispositivo Windows

| Escenario | Acción recomendada |
|-----------|--------------------|
| Dispositivo perdido o robado | Borrar inmediatamente para proteger los datos de la empresa |
| Offboarding con reasignación del dispositivo | Borrar antes de asignar al nuevo empleado |
| Dispositivo comprometido por malware | Borrar y reinstalar un sistema operativo limpio |
| Renovación rutinaria de dispositivos | El borrado es opcional — depende de su política de seguridad |

## Borrar un dispositivo Windows

1. Vaya a **Dispositivos** y localice el dispositivo que desea borrar.
2. Abra la página de detalles del dispositivo.
3. Haga clic en **Acciones** > **Borrar**.
4. Confirme la acción en el cuadro de diálogo de confirmación.

El dispositivo recibe el estado **Borrado pendiente**. Una vez que el dispositivo confirma el comando, el estado se actualiza a **Borrado**.

## Después del borrado

Una vez que un dispositivo ha sido borrado, siga estos pasos antes de volver a inscribirlo:

1. **Reinstalar Windows** — dado que `doWipeProtected` limpia completamente el disco, es posible que necesite reinstalar el sistema operativo usando una unidad USB de recuperación o arranque por red (PXE).
2. **Eliminar el host de Fleet** — antes de volver a inscribir el dispositivo, elimine su registro de Fleet. Esto garantiza que las claves de cifrado del disco se depositen correctamente en la nueva inscripción.
3. **Volver a inscribir el dispositivo** — siga su proceso de inscripción estándar para volver a poner el dispositivo bajo gestión.

<Note>
  Si el dispositivo estaba inscrito mediante Windows Autopilot y todavía está registrado en su tenant de Azure/Entra, entrará automáticamente en el flujo de Autopilot en la próxima configuración. Consulte [inscripción zero-touch Windows](/es/mdm/zero-touch/zero-touch-windows) para más detalles.
</Note>

## Limitaciones

- **Windows Home no es compatible** — el borrado remoto requiere Windows Pro, Enterprise o Education. Consulte [gestionar dispositivos Windows Home](/es/mdm/policies/manage-windows-home) para alternativas.
- **El dispositivo puede no arrancar después del borrado** — en algunas configuraciones de hardware, `doWipeProtected` puede dejar el dispositivo sin capacidad de arranque. Tenga una unidad USB de recuperación preparada.
- **El borrado es irreversible** — una vez iniciado y confirmado por el dispositivo, el borrado no puede cancelarse. Todos los datos se eliminarán permanentemente.
- **Clave de recuperación de BitLocker** — si BitLocker estaba activado, asegúrese de que la clave de recuperación esté respaldada antes del borrado. Después del borrado, la clave anterior ya no es válida.
