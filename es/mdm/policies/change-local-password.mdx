Este artículo explica cómo actualizar una contraseña de sesión local desde la cabina Primo. Esta acción permite a los administradores actualizar o restablecer de forma remota las contraseñas de los usuarios en los dispositivos administrados.

## Requisitos previos

- Eres administrador del espacio de tu empresa.
- El dispositivo está registrado en MDM.
- El dispositivo está **En línea** (encendido y conectado a Internet).
- Existe una sesión de administrador en el dispositivo.

## Actualiza la contraseña de Primo

1. En la página de detalles del dispositivo, vaya a la pestaña **Usuarios**.
2. Haga clic en **Acción** junto a la sesión que desea modificar.
3. Ingrese la nueva contraseña temporal y guárdela.

Puede monitorear el estado del cambio de contraseña en la pestaña **Registros**.

## Utilice la nueva contraseña en el dispositivo

Usted o el usuario pueden iniciar sesión con las nuevas credenciales en la siguiente sesión.

La contraseña de la sesión local ahora está actualizada.

## Restablecer el PIN de Windows Hello

Si el código PIN de Windows Hello no funciona y el usuario no puede usar su contraseña, puede restablecer el PIN ejecutando un script de PowerShell que elimina la carpeta PIN de Windows Hello (NGC) y reinicia el servicio relacionado:

- Este script detiene el servicio NgcCtnrSvc.
- Toma posesión y establece permisos en la carpeta NGC.
- Elimina el contenido de la carpeta NGC para forzar la recreación del PIN.
- Reinicia el servicio NgcCtnrSvc.

Se le pedirá al usuario que cree un nuevo PIN después del restablecimiento.

**Importante:**
- Importe y ejecute el script con Fleet
- Lo ideal es ejecutar el script cuando el usuario cierre la sesión.
- Los registros de esta operación se guardan en `C:\ProgramData\MDM\Logs\Reset-WindowsHelloPIN.log`.
