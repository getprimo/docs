Integra Okta come provider di autenticazione per i tuoi dispositivi gestiti utilizzando la piattaforma Single Sign-On di Apple. Una volta configurati, i dipendenti accedono al proprio Mac con le credenziali Okta (non è richiesta alcuna password locale separata) e l'SSO si propaga a tutte le app connesse a Okta.

| ✅ | | | | |

## Su macOS

### Aggiungi l'applicazione Platform Single Sign-On in Okta

1. Vai a **Console di amministrazione > Applicazioni > Applicazioni > Sfoglia il catalogo delle app**
2. Cerca **Single Sign-On della piattaforma per macOS**
3. Fai clic su **Aggiungi integrazione**

   > Se viene visualizzato il messaggio *Questa funzione non è abilitata*, contatta il tuo account manager Okta per abilitare l'accesso al dispositivo Okta (funzione a pagamento).
4. Apri l'app dall'elenco delle applicazioni
5. Nella scheda **Generale**, rinomina l'etichetta dell'app, se necessario
6. Nella scheda **Accedi**, prendi nota dell'**ID client** (richiesto per la configurazione MDM)
7. Nella scheda **Assegnazioni**, assegna l'app agli utenti o ai gruppi pertinenti

### Crea una sfida SCEP in Okta

1. Vai a **Console di amministrazione > Sicurezza > Integrazioni dispositivo**
2. Assicurati che la scheda **Accesso al dispositivo** sia visibile (accanto a *Gestione endpoint*)

   > Se la scheda manca, Okta Device Access non è ancora abilitato
3. Aggiungi un'**Autorità di certificazione SCEP statica**
4. Prendere nota di quanto segue:
   - **URL del certificato SCEP**
   - **Sfida per il certificato SCEP**

### Aggiungi Okta Verifica a Fleet

1. Dalla console di amministrazione Okta, vai su **Impostazioni > Download**
2. Scarica **Okta Verifica per macOS**
3. Aggiungi l'app a Fleet come software:
   - Basta caricare il file dell'applicazione
   - Non abilitare la distribuzione automatica o il self-service
   - **Primo regolerà automaticamente le impostazioni per gestire la distribuzione**

### Conserva le informazioni chiave per la configurazione di MDM

Assicurati di salvare i seguenti dettagli per i prossimi passaggi di configurazione:

- **Dominio Okta** (ad es. `yourorganization.okta.com`)
- **ID cliente** (dall'applicazione Single Sign-On della piattaforma)
- **URL del certificato SCEP**
- **Sfida per il certificato SCEP**

### Finalizzare la distribuzione

Per finalizzare la distribuzione di **Okta Platform SSO**, vai a un profilo: https://app.getprimo.com/profiles

1. Abilita l'impostazione **Okta SSO piattaforma**
2. Inserisci le informazioni richieste (dominio, ID cliente, URL SCEP e Challenge)
3. Salva le modifiche

L'SSO della piattaforma Okta verrà quindi attivato su tutti i dispositivi target del profilo.

<Note>
  Il completamento della distribuzione potrebbe richiedere diversi minuti, poiché Okta Verify deve essere installato su tutti i dispositivi di destinazione.
</Note>
