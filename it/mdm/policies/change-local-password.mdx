Questo articolo spiega come aggiornare una password di sessione locale dal abitacolo Primo. Questa azione consente agli amministratori di aggiornare o reimpostare in remoto le password degli utenti sui dispositivi gestiti.

## Prerequisiti

- Sei un amministratore del tuo spazio aziendale.
- Il dispositivo è registrato in MDM.
- Il dispositivo è **Online** (acceso e connesso a Internet).
- Sul dispositivo esiste una sessione di amministratore.

## Aggiorna la password da Primo

1. Nella pagina dei dettagli del dispositivo, vai alla scheda **Utenti**.
2. Fai clic su **Azione** accanto alla sessione che desideri modificare.
3. Inserisci la nuova password temporanea e salvala.

Puoi monitorare lo stato di modifica della password nella scheda **Log**.

## Utilizza la nuova password sul dispositivo

Tu o l'utente potrete accedere con le nuove credenziali all'accesso alla sessione successiva.

La password della sessione locale è ora aggiornata.

## Reimpostazione del PIN di Windows Hello

Se il codice PIN di Windows Hello non funziona e l'utente non può utilizzare la propria password, puoi reimpostare il PIN eseguendo uno script PowerShell che elimina la cartella PIN di Windows Hello (NGC) e riavvia il relativo servizio:

- Questo script arresta il servizio NgcCtnrSvc.
- Assume la proprietà e imposta le autorizzazioni sulla cartella NGC.
- Rimuove il contenuto della cartella NGC per forzare la ricreazione del PIN.
- Riavvia il servizio NgcCtnrSvc.

All'utente verrà richiesto di creare un nuovo PIN dopo il ripristino.

**Importante:**
- Importa ed esegui lo script con Fleet
- Idealmente, eseguire lo script quando l'utente è disconnesso.
- I registri di questa operazione vengono salvati su `C:\ProgramData\ZXPROTECT0ZX\Logs\Reset-WindowsHelloPIN.log`.
