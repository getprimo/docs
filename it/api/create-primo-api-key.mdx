Primo Le chiavi API vengono utilizzate per autenticare le richieste a [Primo REST API](https://api.getprimo.com/openapi.json). Esistono due tipi di chiavi API, ciascuna con un ambito diverso.

## Tipi di chiave API

### Chiave API dell'azienda

Una **chiave API aziendale** ha come ambito una singola azienda. Fornisce l'accesso a tutti gli endpoint che operano su risorse a livello aziendale: dispositivi, dipendenti, accessori, ticket, SaaS e altro ancora.

### Tasto API dell'area di lavoro

Una **chiave API dell'area di lavoro** ha come ambito l'intera area di lavoro. Fornisce accesso a endpoint a livello di area di lavoro, come l'elenco e la visualizzazione delle aziende all'interno dell'area di lavoro.

<Note>
La maggior parte degli endpoint API richiede una **chiave API aziendale**. Solo gli endpoint specifici dell'area di lavoro (in `/workspace/`) richiedono una **chiave API dell'area di lavoro**. Controlla il riferimento API per il badge su ciascun endpoint.
</Note>

## Genera una chiave API aziendale

1. Vai a **Impostazioni** > **API**.
2. Fare clic su **Crea chiave API**.
3. Assegna un nome alla chiave (ad esempio "La mia integrazione").
4. Fare clic su **Crea**.
5. Copia immediatamente la chiave: non verrà più visualizzata.

<Warning>
Conserva la chiave API in modo sicuro. Chiunque disponga di questa chiave può accedere ai dati Primo tramite API.
</Warning>

## Utilizzare la chiave API

Passa la chiave nell'intestazione `Authorization` delle tue richieste:

```bash
curl https://api.getprimo.com/v1/devices \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Revoca una chiave

Per revocare una chiave, vai su **Impostazioni** > **API**, trova la chiave e fai clic su **Elimina**.
È inoltre possibile eliminare una chiave API revocata.
