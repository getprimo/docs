Le chiavi Primo API vengono utilizzate per autenticare le richieste a [Primo REST API](https://api.getprimo.com/openapi.json). Esistono due tipi di chiavi API, ciascuna con un ambito diverso.

## Tipi di chiavi API

### Chiave API Azienda

Una **chiave API Azienda** ha come ambito una singola azienda. Fornisce accesso a tutti gli endpoint che operano sulle risorse a livello aziendale — dispositivi, dipendenti, accessori, ticket, SaaS e altro.

### Chiave API Workspace

Una **chiave API Workspace** ha come ambito l'intero spazio di lavoro. Fornisce accesso agli endpoint a livello di workspace, come l'elenco e la visualizzazione delle aziende nel tuo workspace.

<Note>
La maggior parte degli endpoint API richiede una **chiave API Azienda**. Solo gli endpoint specifici del workspace (sotto `/workspace/`) richiedono una **chiave API Workspace**. Consulta il riferimento API per il badge su ogni endpoint.
</Note>

## Genera una chiave API Azienda

1. Vai a **Impostazioni** > **API**.
2. Fare clic su **Crea chiave API**.
3. Assegna un nome alla chiave (ad esempio "La mia integrazione").
4. Fare clic su **Crea**.
5. Copia immediatamente la chiave: non verrà più visualizzata.

<Warning>
Conserva la chiave API in modo sicuro. Chiunque abbia questa chiave può accedere ai tuoi dati Primo tramite API.
</Warning>

## Utilizzare la chiave API

Passa la chiave nell'intestazione `Authorization` delle tue richieste:

```bash
curl https://api.getprimo.com/v1/devices \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Revoca una chiave

Per revocare una chiave, vai su **Impostazioni** > **API**, trova la chiave e fai clic su **Elimina**.
È inoltre possibile eliminare una chiave API revocata.
