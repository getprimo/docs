---
title: "Configurer le provisionnement des licences"
description: "Configurez le provisionnement pour automatiser la création et les mises à jour de comptes utilisateur dans des applications telles que Zendesk, en garantissant que les employés disposent de rôles d'accès appropriés basés sur des règles définies sans gestion manuelle."
---

Le provisionnement garantit que les comptes d'utilisateurs sont automatiquement créés et mis à jour dans les applications connectées selon les règles que vous définissez. En configurant le provisionnement, vous pouvez attribuer des applications et des rôles spécifiques à des groupes ciblés d'employés.

Cet article explique comment configurer le provisionnement à l'aide de règles.

## Règles d'accès

1. Dans la console Primo, accédez à **Identités et accès**.
2. Sélectionnez **Règles**.
3. Soit **mettre à jour une règle existante**, soit **créer une nouvelle** pour gérer les règles de provisionnement.

## Créer de nouvelles règles (facultatif)

![Capture d'écran : Règles d'accès](/images/notion-import/set-up-licence-provisioning__img-090__block-0008__25443d9f.png)


Si vous devez cibler un nouveau groupe de collaborateurs, créez une règle dédiée :

1. Cliquez sur **Créer une nouvelle règle**.
2. Saisissez un nom pour la règle (par exemple, *Dev Team*).
3. Définissez les conditions de ciblage sous **Employés ciblés** (par exemple : *L'équipe est en développement*).
4. Enregistrez votre règle.
    ![Capture d'écran : Créer de nouvelles règles (facultatif)](/images/notion-import/set-up-licence-provisioning__img-091__block-0016__25443d9f.png)


## Activer le provisionnement pour une application

![Capture d'écran : Créer de nouvelles règles (facultatif)](/images/notion-import/set-up-licence-provisioning__img-092__block-0017__25443d9f.png)


1. Ouvrez la règle que vous souhaitez utiliser.
2. Accédez à l'onglet **Applications**.
3. Sélectionnez l'application que vous avez déjà connectée (par exemple, Zendesk).
4. Activez **Provisionnement** pour l'activer.
5. Sous **Sélection des rôles**, choisissez le rôle que vous souhaitez attribuer aux employés ciblés.

Par exemple, pour attribuer le rôle *Agent léger* dans Zendesk à votre équipe de développement :

- Créez ou modifiez la règle *Dev Team*.
- Définir le ciblage : *L'équipe est le développement*.
- Sélectionnez **Applications > Zendesk**.
- Activez **Provisioning** et attribuez le rôle *Light agent*.
    ![Capture d'écran : Activer le provisionnement pour une application](/images/notion-import/set-up-licence-provisioning__img-093__block-0030__25443d9f.png)


## Résultat

Une fois le provisionnement activé et enregistré :

- Les utilisateurs répondant aux conditions de ciblage de la règle sont automatiquement créés dans l'application s'ils ne sont pas déjà présents.
- Les rôles attribués (par exemple, Zendesk *Light agent*) sont appliqués automatiquement.
- Toute modification apportée aux utilisateurs dans le système RH sera synchronisée avec l'application via Primo.

Cela garantit que les employés disposent toujours du bon niveau d’accès sans gestion manuelle des comptes.
