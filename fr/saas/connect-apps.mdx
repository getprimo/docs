La solution de gestion SaaS permet à votre organisation de gérer l'accès aux applications métiers de manière sécurisée et centralisée. En connectant vos applications, vous garantissez que les utilisateurs sont automatiquement provisionnés, que l'authentification est unifiée et que les droits d'accès sont appliqués de manière cohérente dans tous vos outils.

Cet article explique les étapes nécessaires pour connecter une application.

## Cartographie automatisée SaaS

Si vous utilisez Google Workspace, vous pouvez le connecter dans Identités & Accès. Il mappera automatiquement toutes les applications pour lesquelles votre Google Workspace est utilisé comme connecteur SSO.

![Capture d'écran : [à venir] Mappage Saas automatisé](/images/notion-import/connect-your-apps__img-086__block-0005__27343d9f.png)


Une fois connecté, vous pouvez voir toutes les identités liées à chaque application et définir un propriétaire spécifique.

![Capture d'écran : [à venir] Mappage Saas automatisé](/images/notion-import/connect-your-apps__img-087__block-0007__27343d9f.png)


## Accéder au catalogue d'applications

1. Connectez-vous au cockpit Primo.
2. Dans le menu de navigation, accédez à **SaaS**.
3. Sélectionnez **Identités** et **Ajouter un SaaS**.

Le [Catalogue d'applications](https://app.getprimo.com/saas-applications/add-app) contient une liste d'applications directement intégrables.

## Sélectionnez et configurez l'application

![Capture d'écran : Accéder au catalogue d'applications](/images/notion-import/connect-your-apps__img-088__block-0013__25443d9f.png)


1. Localisez l'application à laquelle vous souhaitez vous connecter.
2. Cliquez sur l'application pour ouvrir sa page de configuration.
3. Suivez les instructions étape par étape fournies.

Les étapes de configuration varient selon le type d'application :

- **Single Sign-On (SSO)** : vous devrez peut-être fournir des métadonnées ou des certificats pour établir la confiance avec l'application.
- **Provisionnement SCIM** : vous devrez peut-être saisir les informations d'identification API pour autoriser la création et les mises à jour automatiques de comptes utilisateur.
- **Intégrations basées sur API** : certaines applications nécessitent que vous génériez et saisissiez des clés API.

Des conseils à l’écran sont fournis pour chaque application pour terminer la configuration.

![Capture d'écran : Sélectionner et configurer l'application](/images/notion-import/connect-your-apps__img-089__block-0024__25443d9f.png)


## Conclusion

Vous pouvez cliquer sur l'onglet Journaux et vérifier que la synchronisation est active.

Si vous pensez que votre forfait est insuffisant pour configurer votre application, vous pouvez utiliser la connexion basée sur l'IA.
