Depuis **MDM > Logiciel**, vous pouvez déployer des applications sur vos appareils macOS, Windows et Linux gérés sans jamais quitter Primo. Primo prend en charge quatre types d'applications :

- **Fleet-Maintained Apps (FMA)** — un catalogue organisé d'applications populaires maintenu et mis à jour par Fleet
- **Apple App Store (macOS)** : applications macOS achetées via le programme d'achat en volume (VPP) d'Apple
- **Applications personnalisées** : vos propres packages `.pkg`, `.msi`, `.deb` ou `.rpm`

<Note>
Pour le déploiement d'applications iOS/iPadOS et Android, voir [Déployer des applications (iOS et Android)](/fr/mdm/policies/manage-apps-smartphone).
</Note>

---

## Options de déploiement

Lors de l'ajout d'une application, vous pouvez configurer les options suivantes :

### Installer automatiquement
Lorsqu'elle est activée, l'application est automatiquement transmise à tous les appareils correspondants. Si cette case n'est pas cochée, l'application ne sera installée que lorsque vous déclencherez manuellement l'installation sur un appareil spécifique.

### Mise à disposition via Self-Service
Lorsqu'elle est activée, l'application apparaît dans le portail **Self-Service**, accessible aux collaborateurs via l'agent Fleet. Les collaborateurs peuvent l'installer eux-mêmes sur demande.

### Installer pendant Zero Touch
Lorsqu'elle est activée, l'application est installée dans le cadre du flux de déploiement Zero Touch. Cela garantit que les applications principales sont présentes avant que l'appareil ne soit remis à l'employé.

<Warning>
L'activation de **Installer pendant Zero Touch** peut augmenter le temps de provisionnement. Réservez-le aux applications réellement nécessaires avant la première utilisation.
</Warning>

## Correctifs d'application

Pour les types d'applications éligibles, Primo propose deux bascules de mise à jour automatique :

| Basculer | Ce que ça fait |
|--------|--------------|
| **Maintenir le programme d'installation des sources à jour** | Vérifie une fois par jour une version plus récente du programme d'installation et la remplace automatiquement |
| **Mettre à jour l'application sur tous les appareils** | Met automatiquement à jour l'application sur tout appareil sur lequel la version installée est plus ancienne que le programme d'installation actuel |

## Ajouter une application gérée par Fleet (FMA)

Les applications Fleet-Maintained sont une bibliothèque organisée de logiciels populaires (par exemple Chrome, Zoom, Slack) que Fleet maintient à jour.

1. Accédez à **MDM > Logiciel** et cliquez sur **Ajouter une application**.
2. Sélectionnez **Application maintenue par Fleet**.
3. Choisissez l'application dans la liste.
4. Configurez les options de déploiement : **Installer automatiquement**, **Self-Service** et/ou **Zero Touch**.
5. Activez les boutons de correctifs si nécessaire.
6. Cliquez sur **Ajouter un logiciel**.

## Ajouter une application Apple App Store (macOS VPP)

<Note>
**Prérequis** : Vous devez disposer d'un compte Apple Business Manager (ABM) avec configuration VPP. Voir [Déployer des applications VPP](/fr/mdm/policies/deploy-vpp-applications) pour les instructions de configuration.
</Note>

1. Accédez à **MDM > Logiciel** et cliquez sur **Ajouter une application**.
2. Sélectionnez **App Store (VPP)**.
3. Parcourez ou recherchez l’application macOS.
4. Configurez les options de déploiement : **Installer automatiquement**, **Self-Service** et/ou **Zero Touch**.
5. Cliquez sur **Ajouter un logiciel**.

<Note>
Le programme d'installation source provient toujours directement de l'App Store d'Apple. L'option **Maintenir le programme d'installation source à jour** n'est pas applicable. La mise à jour automatique n'est pas disponible pour les applications macOS VPP.
</Note>

## Ajouter une application personnalisée

Les applications personnalisées sont des packages que vous téléchargez vous-même (`.pkg` pour macOS, `.msi` pour Windows, `.deb`/`.rpm` pour Linux).

1. Accédez à **MDM > Logiciel** et cliquez sur **Ajouter une application**.
2. Sélectionnez **Application personnalisée**.
3. Téléchargez votre fichier de package.
4. Configurez les options de déploiement : **Installer automatiquement**, **Self-Service** et/ou **Zero Touch**.
5. Cliquez sur **Ajouter un logiciel**.

<Warning>
**La prise en charge de l'installation automatique varie selon le type de fichier.** Les types suivants ne prennent pas en charge l'option **Installer automatiquement** :
- `.exe` (Windows)
- `.sh` (Linux)
- `.ps1` (Windows PowerShell)

Utilisez `.msi` pour Windows et `.pkg` pour macOS pour activer le déploiement automatique.
</Warning>

<Note>
La mise à jour automatique des applications personnalisées **arrivera bientôt**. L'option **Maintenir le programme d'installation des sources à jour** n'est pas disponible pour les applications personnalisées.
</Note>

## Supprimer une application

### Désactiver l'installation automatique

Accédez à **MDM > Logiciel**, recherchez l'application et décochez **Installer automatiquement**. Cela arrête les futures installations automatiques mais ne supprime pas l'application des appareils sur lesquels elle est déjà installée.

### Désinstaller à partir d'un appareil spécifique

1. Accédez à **MDM > Appareils** et ouvrez l'appareil.
2. Ouvrez l'onglet **Logiciel**.
3. Recherchez l'application et cliquez sur **Désinstaller**.
