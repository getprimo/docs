Cet article explique comment mettre à jour un mot de passe de session locale depuis le cockpit Primo. Cette action permet aux administrateurs de mettre à jour ou de réinitialiser à distance les mots de passe des utilisateurs sur les appareils gérés.

## Conditions préalables

- Vous êtes administrateur de votre espace entreprise.
- L'appareil est inscrit dans le MDM.
- L'appareil est **En ligne** (allumé et connecté à Internet).
- Une session administrateur existe sur l'appareil.

## Mettre à jour le mot de passe de Primo

1. Sur la page des détails de l'appareil, accédez à l'onglet **Utilisateurs**.
2. Cliquez sur **Action** à côté de la session que vous souhaitez modifier.
3. Saisissez le nouveau mot de passe temporaire et enregistrez-le.

Vous pouvez surveiller l'état du changement de mot de passe dans l'onglet **Journaux**.

## Utilisez le nouveau mot de passe sur l'appareil

Vous ou l'utilisateur pouvez vous connecter avec les nouvelles informations d'identification lors de la prochaine connexion à la session.

Le mot de passe de la session locale est désormais mis à jour.

## Réinitialisation du code PIN Windows Hello

Si le code PIN Windows Hello ne fonctionne pas et que l'utilisateur ne peut pas utiliser son mot de passe, vous pouvez réinitialiser le code PIN en exécutant un script PowerShell qui supprime le dossier PIN Windows Hello (NGC) et redémarre le service associé :

- Ce script arrête le service NgcCtnrSvc.
- Prend possession et définit les autorisations sur le dossier NGC.
- Supprime le contenu du dossier NGC pour forcer la recréation du code PIN.
- Redémarre le service NgcCtnrSvc.

L'utilisateur devra créer un nouveau code PIN après la réinitialisation.

**Important:**
- Importez et exécutez le script avec Fleet
- Idéalement, exécutez le script lorsque l'utilisateur est déconnecté.
- Les journaux de cette opération sont enregistrés dans `C:\ProgramData\MDM\Logs\Reset-WindowsHelloPIN.log`.
