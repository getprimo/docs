<Note>
  **Type de déploiement : Automatique**

  SentinelOne est déployé **automatiquement** via MDM. Une fois que vous avez connecté votre instance SentinelOne et créé un contrôle MDM, l'agent est transmis à tous les appareils ciblés sans aucune étape manuelle. Aucune interaction de l'utilisateur n'est requise.
</Note>

Cet article explique comment activer l'intégration de SentinelOne dans Primo pour déployer automatiquement l'agent sur tout ou partie de votre parc d'appareils.

## Conditions préalables

- Avoir Primo MDM
- Avoir une licence SentinelOne (achetée via Primo ou existante)

<Note>
  Besoin d'une instance SentinelOne ? Contactez-nous pour plus d'informations.
</Note>

## Activer l'intégration de SentinelOne

1. Vous pouvez activer l'intégration de SentinelOne à partir de vos **Paramètres \> MDM \> Intégrations (héritées)**
2. Remplissez le formulaire avec les informations suivantes :
   - **URL SentinelOne** : `https://xxxxxxxxx.sentinelone.net`
   - **Identifiant du site** :
     - Dans SentinelOne, ouvrez **Politique et paramètres \> Sites**.
     - Cliquez sur votre site pour afficher l'ID du site.
   - **Jeton d'utilisateur de service** :
     - Dans SentinelOne, ouvrez **Politique et paramètres \> Utilisateurs du service**.
     - Créez un nouvel utilisateur de service (**Nouvel utilisateur de service**).
     - Sélectionnez votre site et attribuez le rôle **Admin**.

## Déployez SentinelOne sur votre flotte

Le déploiement de SentinelOne sur les appareils de votre flotte se fait via vos configurations MDM.

1. Dans Primo, accédez à **MDM \> Contrôles \> Ajouter un nouveau contrôle**
2. Choisissez la plateforme (plateformes disponibles)
3. Activer la carte **EDR - SentinelOne**
4. Mettez à jour le ciblage pour cibler les appareils souhaités
5. Confirmer

L'agent SentinelOne sera automatiquement installé sur tous les appareils inclus dans le contrôle.

## Migrer SentinelOne

Si vous devez migrer vos points de terminaison d'une console S1 à une autre, vous devrez suivre les étapes suivantes :

- Sur votre instance Fleet, mettez à jour les scripts d'installation des packages Sentinel Agent avec le nouveau jeton de site.
  - Accédez à votre instance de flotte : https://xxx.mdm.getprimo.com
  - Ouvrez l'onglet Logiciel
  - Recherchez « Sentinel » (SentinelOne, Sentinel Agent, etc.)
  - Sélectionnez chaque forfait
  - Cliquez sur le bouton Modifier
  - Cliquez sur "options avancées"
  - Mettre à jour le script d'installation
- Sur votre console S1, sélectionnez tous les points de terminaison que vous souhaitez migrer et cliquez sur Actions \> Actions sur les points de terminaison \> migrer les points de terminaison.
- Saisissez le nouveau token du site et validez.

## Plateformes disponibles

SentinelOne dans Primo est actuellement disponible pour les plateformes suivantes :

- **macOS** (arm64) - Puces M1 et plus
- **Windows** (x64) - Prend en charge Windows Pro
- **Linux**
  - Packages .deb (arm64) - Compatible avec les distributions Ubuntu, Debian et dérivées
  - Packages .rpm (arm64) - Compatible avec les distributions RHEL, CentOS, Fedora, SUSE et dérivées

Si la plate-forme n'est pas mentionnée ci-dessus, vous pouvez toujours installer manuellement SentinelOne en téléchargeant le programme d'installation souhaité sur le site Web SentinelOne et en l'installant manuellement avec Fleet.
